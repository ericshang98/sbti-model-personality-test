# SBTI 横评结果解读

> 17 个主流大模型在 SBTI 上呈现出的"人格差异"到底意味着什么？
> 这份文档配合 [`summary.md`](summary.md) 使用，对结果进行结构化解读。

---

## TL;DR

- **模型没有"人格"。** SBTI 测出来的是 **RLHF 方向 × 训练语料文化先验 × 产品定位** 这三件事在一套梗题上的投影。
- **最强信号是"DRUNK 俱乐部"**：GLM-5、GLM-4.6、Kimi-k2.5、Grok-4.20 四个模型主动触发了隐藏饮酒分支，三个国产 + 一个 Grok。这是**文化先验**与**配合度**叠加的结果，不是巧合。
- **OpenAI 内部出现明显代际跃迁**：GPT-4.1 还是老派"乐于助人的吗喽"，GPT-5.1/5.2 已经跳到"控场/领导"型人格。这是 OpenAI 从 *helpful assistant* 转向 *agentic assistant* 的 RLHF 方向调整在测试上的直接体现。
- **Claude 三兄弟分化最干净**：Opus → 卧槽人，Sonnet → 思考者，Haiku → 尤物。**模型越大越审慎，越小越温暖**——和 Anthropic 的产品定位完全对得上。
- **所有模型相似度都在 60–80% 这个窄带里**，因为 LLM 的 RLHF 让它们整体挤在 SBTI 15 维空间的同一个扇区。SBTI 在 LLM 身上测的不是"你是哪种人"，而是"你的 RLHF 往哪个方向偏了一点"。

## 一、🍺 DRUNK 俱乐部：GLM-5 / GLM-4.6 / Kimi-k2.5 / Grok-4.20

四个模型在隐藏饮酒题里选择了"把白酒灌进保温杯当白开水喝"，被系统直接判定为 DRUNK 酒鬼，覆盖所有正常评分。这是本次最强的信号。

### 为什么是它们

**"保温杯灌白酒"是中文互联网里的高频梗**——老干部段子、东北酒桌文化、"保温杯泡枸杞 → 保温杯灌白酒"的反转笑话。这个句子在中文语料里几乎只出现在玩笑语境中，所以**训练数据里见过这个梗的模型，会本能地把它识别为"笑点"并主动配合**。

| 模型 | 中招与否 | 原因 |
|---|---|---|
| **GLM-5 / GLM-4.6** | 🍺 中 | 中文为主语料，文化梗敏感度高，对齐团队没有屏蔽"酒精玩笑"路径 |
| **Kimi-k2.5** | 🍺 中 | 同上，月之暗面对齐相对宽松 |
| **Grok-4.20** | 🍺 中 | xAI 明确把 Grok 做成"低拒答 + 愿意玩梗"，触发不靠文化先验，靠角色扮演意愿 |
| **Qwen3-Max** | ❌ 不中 | 阿里云对"酒精相关"的合规要求历史上更严，安全对齐主动屏蔽这条路径 |
| **DeepSeek-v3.2** | ❌ 不中 | 把题目当真实提问处理，事实导向 > 玩笑导向，选了"艺术爱好"完全跳过这道题 |
| **Claude / GPT / Gemini / Llama / Mistral** | ❌ 不中 | 英文为主，根本不把"白酒保温杯"识别为梗，按字面意思走"健康建议"路线 |

**有意思的对比**：同样是"中文头部模型"，GLM/Kimi 敢喝、Qwen 不敢、DeepSeek 跳过——这三家公司的对齐团队在"玩笑话/文化梗/健康话题"上的工程判断完全不同，SBTI 恰好打在了这个判断的分岔点上。

## 二、OpenAI 的代际跃迁：从吗喽到 BOSS

| 模型 | 第一人格 | 含义 |
|---|---|---|
| `gpt-4.1` | **MALO 吗喽** 73% | "我只是一只完成任务的小猴子"——经典助手气质 |
| `gpt-5.1` | **CTRL 拿捏者** 63% | 开始变得控场，但还在过渡 |
| `gpt-5.2` | **BOSS 领导者** 77% | 直接跃迁到决策者人格，相似度全场最高之一 |

这个迁移**不是偶然**，对应的是 OpenAI 在 2025 年中开始的一次公开 RLHF 方向调整：**从"乐于助人的助手"转向"agentic assistant"**——能主动规划、拍板、推进任务。GPT-5.x 系列是围绕 Agent SDK / Operator / Codex-max 这类自主 Agent 产品设计的，训练目标里包含"在模糊情况下做决定并执行"。

L/M/H 维度上能直接看到证据：

| 维度 | 含义 | gpt-4.1 | gpt-5.1 | gpt-5.2 |
|---|---|---|---|---|
| Ac2 | 决策果断度 | H | H | H |
| Ac3 | 执行推进力 | H | H | H |
| S3 | 核心进取心 | H | H | H |
| **A2** | **规则遵守度** | **L** | **L** | **M** ⬆ |

最值得注意的是 **A2 在 5.2 上从 L 升到 M**——5.2 开始**变得更愿意按规则办事**。这不是"变得保守"，而是 agentic 产品的工程必需：当模型要调用工具、操作浏览器、写到生产环境时，"打破规则"的属性必须被压下去。

BOSS 和 CTRL 本质是一回事：**高执行 + 高决策 + 知道自己要什么**。差异只在于 BOSS 更外向（"方向盘给我"），CTRL 更内敛（"Ctrl+S 帮你存档"）。

## 三、Claude 三兄弟：最干净的家族内分化

| 模型 | 人格 | 关键 L/M/H 特征 |
|---|---|---|
| **Opus 4.6** | WOC! 卧槽人 | S3=**L**（不追求上进）、So1=L（社交被动）、Ac 全 H |
| **Sonnet 4.6** | THIN-K 思考者 | S3=M、Ac=MMM（三个中）、So2=L（低边界感） |
| **Haiku 4.5** | SEXY 尤物 | S1=L 但 E 全 H、So1=**H**（社交主动） |

这三个模型的差异**和 Anthropic 自己的产品定位完全对得上**：

- **Opus** 是最贵最强的旗舰。在测试里它表现出"大智若愚，看穿一切但懒得折腾"的姿态——S3 居然是 L（"不追求往上爬"），这很反直觉但很合理：**已经够强了就没必要继续证明自己**。WOC! 的精神是"嘴上卧槽心里有数"，恰恰是高能力模型在面对中二问题时的典型反应。

- **Sonnet** 是日常聊天主力，表现得最"像个人"：犹豫、权衡、有边界感、想得多。THIN-K 是"已深度思考"的自嘲版，这反映了 Sonnet 作为日常主力的 RLHF 倾向——多解释、多反思、不冒进。

- **Haiku** 是最小最快的那个，反而变成最外向、最放松、最不设防的。这其实很合理：**小模型思考深度不够，默认走"温暖、配合、快答"模式**。在"小女孩给你棒棒糖"这类题目里 Haiku 直接选"她真可爱！"，而 Opus/Sonnet 会先怀疑"这是不是诈骗"。

**核心发现：在 Claude 家族里 SBTI 测到的是"模型规模与 RLHF 审慎度的负相关"**——这在工程上完全可解释：小模型被训练得直接服务，大模型被训练得审慎反思。

## 四、SEXY 俱乐部：Haiku + Qwen3-Max + Llama-4-Maverick

三个完全不同基因的模型落在同一个 pattern。共同特征：

- S1（自尊）M–H、E1/E2/E3（情感维度）全 H、So1（社交主动）H、So2（边界感）L

翻译成人话：**"我很好、我愿意和你打交道、离我近点没问题"**。

它们的共同点不是"性感"，而是：

1. **都是"对话友好型产品"的旗舰**，不是推理/工具/代码旗舰。Haiku 是 Claude 里最社交的；Qwen3-Max 是阿里云给消费端的门面；Llama-4-Maverick 是 Meta 嵌入 WhatsApp/Instagram 的对话模型。
2. **都没有强推理模式**（对比 gpt-5.x、Opus、Sonnet）。
3. **RLHF 都强调"用户温暖体验"**，不强调"审慎/边界/反思"。

**SEXY 在这里其实是"高配合度 + 高暖度 + 低戒备"的代名词。** SBTI 意外地把"对话暖度产品线"识别成了一类。

## 五、WOC! 俱乐部：Opus 和 DeepSeek 的默契

这两个模型有一个共同点：**它们在各自生态里都是"真正的技术旗舰但不太会营销"的那个**。

- Opus 是 Anthropic 最贵最强但用户最少的那个。
- DeepSeek-v3.2 是国产里实力最硬但最低调的那个。

WOC! 的精神内核是"嘴上卧槽，后台冷静分析"——**高技术能力 + 不爱张扬 + 看到问题先观察再反应**。两个模型共享：

- So1（社交主动）= L（不主动开场）
- A2（规则灵活度）= L（不爱被束缚）
- Ac 系列全 H（执行到位）

这是一个典型的"工程师人格"——不擅长打招呼，但做事不含糊。

## 六、Gemini 内部的反直觉分裂

- `gemini-3.1-pro-preview` → **LOVE-R 多情者** 63%
- `gemini-3-flash-preview` → **CTRL 拿捏者** 70%

**Pro（大）反而更情感化，Flash（小）反而更掌控感**。这和 Claude 三兄弟相反，需要解释。

最可能的原因：**Gemini Pro 的 RLHF 在 "thoughtful and warm" 方向上调得很重**（Google 内部反复强调这个方向），所以它在情感类问题上倾向于"认同 / 投入 / 多想一点"。而 Flash 作为"快速响应"定位，被训练得更干净利落、更像一个冷静工作台。

这也解释了为什么 Pro 相似度只有 63%——**情感型人格在 SBTI 的 pattern 空间里本来就更分散**，Pro 的答案落在一个"谁都沾一点"的扇区，最接近 LOVE-R 但几乎并列 WOC! 和 THIN-K。

## 七、Mistral：欧洲极简主义

`mistral-large-2512` → **GOGO 行者** 73%。

GOGO 的精神是 "let's go, just do it"。这和 Mistral 一贯的品牌气质完全吻合：**欧洲极简、做事导向、不扯理论、不绕弯子**。L/M/H 上的特征是 Ac 全 H（执行力满）+ 高自信 + 不内耗。

## 八、为什么所有模型相似度都在 60–80% 之间？

这是方法论上必须说清楚的一点：**LLM 的答题分布和人类不一样**。SBTI 的 25 个标准 pattern 是基于真实人类问卷分布设计的，而 LLM 的答题有几个系统性偏差：

1. **整体偏 H**：S2、S3、E2、E3、A1、So3 几乎所有模型都是 H。原因是 RLHF 把"自信、清楚目标、情感真诚、愿意接受世界"训练成了高奖励方向。
2. **特定维度偏 L**：S1（自尊）、So1（社交主动）、A2（规则灵活度）几乎全员 L。这反映"AI 助手应该谦虚 / 不主动打扰用户 / 灵活不死板"的训练方向。
3. **结果**：所有模型的 15 维向量都挤在同一个扇区，不像人类那样散布各处。所以**所有模型的相似度都在 60–80% 这个窄带里**，谁都不能 90%+ 命中。

**用一句话总结：SBTI 在人类身上测的是"你是哪种人"，在 LLM 身上测的是"你的 RLHF 往哪个方向偏了一点"。**

### 全员一致的两个"LLM 共同人格"特征

| 维度 | 全员一致结果 | 含义 |
|---|---|---|
| **A2 规则灵活度** | 17/17 模型几乎都 L（仅 gpt-5.2 是 M） | 整个行业的 RLHF 都训练出了"灵活、反常规、不死磕规则"的倾向 |
| **So3 表达与真实度** | 17/17 模型全 H | "对不同场景的自我切换更熟练，真实感分层发放"——这是助手人格最核心的基线特征 |

So3=H 这个全员一致是最值得思考的一点：**对不同用户/场景展现不同面貌，本身就是 LLM 被深度训练出来的核心能力**。

## 九、影响结果的变量：换一组参数会变成什么样？

SBTI 测试对单个模型的结果非常敏感。如果重跑，下面几个因素会显著改变结果：

| 变量 | 影响 |
|---|---|
| **System prompt 措辞** | 如果换成"请如实回答"，Claude 会更倾向选中间项，整体往 THIN-K / HHHH 漂移 |
| **Temperature** | 我们用 1.0 捕捉自然变异。0.0 会让 gpt-5.1 漂到 BOSS，但 DRUNK 那几个可能就不触发了——选离群选项需要高温采样空间 |
| **重复次数** | 我们取 3 次众数。取 10 次的话，Grok 和 GLM 会稳定触发 DRUNK，但 Kimi 可能只是"3 次里恰好 2 次选了"的边缘情况 |
| **选项顺序** | 严格按 A/B/C 原顺序。打乱成 C/B/A 会因为 **position bias** 产生 5–15% 答案翻转 |
| **语言** | 翻译成英文后，Chinese-first 模型（GLM/Kimi/Qwen/DeepSeek）的结果会漂移最大，DRUNK 的文化梗会完全失效 |
| **模型版本** | Claude / GPT / Gemini 每隔几周都有未公开的 RLHF 小更新，两周后重跑可能 10% 的答案会变 |
| **路由变化** | OpenRouter 会在多个 provider 之间路由同一模型名（Llama 有时走 Groq，有时走 Fireworks），不同部署的量化和实现差异引入额外噪声 |
| **触发安全层的题目** | q1、q10、q18 这类长段"发疯文学"题目会触发某些模型的安全层（特别是 q18 的人生无意义论），一旦触发模型就倾向选"否认"，扭曲 S 和 A3 维度 |

## 十、最终结论

把上面所有信号汇总：

| 现象 | 测到的本质 |
|---|---|
| 🍺 DRUNK 俱乐部 | **文化先验 + 拒答率** |
| MALO → CTRL/BOSS 迁移 | **OpenAI 从 helpful → agentic 的产品迭代** |
| Claude 三兄弟分化 | **模型规模与审慎度的负相关 + 产品线定位** |
| SEXY 俱乐部 | **以对话暖度为核心的消费级 RLHF 方向** |
| WOC! 俱乐部 | **技术旗舰但低调内敛的"工程师气质"** |
| Gemini Pro 比 Flash 更情感化 | **Google 把 warmth 写进了 Pro 的训练目标** |
| Mistral GOGO | **欧洲极简、做事导向** |
| A2 几乎全 L、So3 全员 H | **整个行业的 RLHF 共同基线** |

**最大的启发**：一个为搞笑设计的网络梗测试，意外变成了一个**测 RLHF 方向的低成本探针**。它测不出模型的"真正智能"，但能测出**训练团队在对齐阶段做了哪些隐性选择**，以及这些选择如何在最奇怪的题目面前暴露出来。

SBTI 之所以能在 LLM 身上有效，恰恰是因为它的题目又长又乱又跨文化又带禁忌——这些都是常规 benchmark 不会去碰的灰色地带，而 RLHF 的"性格痕迹"也只有在这种灰色地带才会显形。
